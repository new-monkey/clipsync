# Security Policy

## Supported Versions

This project currently provides best-effort support for the latest `main` branch and latest tagged release.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Send a private report with:

1. Vulnerability description and impact.
2. Reproduction steps or proof of concept.
3. Affected versions/commit hashes.
4. Suggested mitigation if available.

Contact channel:

- Open a private security advisory on the hosting platform (recommended), or
- Contact repository maintainers directly if private advisories are unavailable.

## Response Process

1. We will acknowledge receipt as soon as possible.
2. We will assess impact and reproduction.
3. We will prepare a fix and coordinate disclosure timing.
4. We will publish a patch release and advisory notes.
